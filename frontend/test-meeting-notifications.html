Meeting Notification Test

This page tests the complete meeting notification flow to ensure participants receive notifications.

Step 1: Create Meeting with Notifications

Create a meeting via the backend API and send notifications to participants.





Step 2: Verify Notifications in localStorage

Check if notifications were properly stored in localStorage for participants.

Step 3: Verify Meeting Visibility

Check if participants can see the meeting in their calendar via API.

Step 4: Manual Test Instructions

To test the complete flow in the actual application:

  1. Login as Admin and go to Calendar
  2. Create a meeting with participants (e.g., users 2, 3, 20)
  3. Check console logs for notification flow
  4. Login as a participant (e.g., user 2)
  5. Check the bell icon for notifications
  6. Verify the meeting appears in their calendar
